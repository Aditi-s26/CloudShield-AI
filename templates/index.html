System Architecture

How CloudShield Works

A three-layer cloud-native architecture combining data ingestion, ML-powered analysis, and intelligent alerting.

Layer 1
📡

Data Collection

Raw network traffic captured and preprocessed using the NSL-KDD benchmark dataset.

  • Network packet capture
  • Feature extraction (41 features)
  • Data normalization
  • Label encoding
Stream
Layer 2
🧠

AI Engine

Random Forest classifier trained on labeled network traffic, detecting anomalies with high precision.

  • Random Forest (100 trees)
  • Multi-class classification
  • Real-time inference
  • Confidence scoring
Alerts
Layer 3
☁️

Cloud Platform

Scalable deployment on AWS/Azure with secure log storage and web-based monitoring dashboard.

  • AWS / Azure deployment
  • Secure cloud log storage
  • Web dashboard (Flask)
  • Auto-scaling enabled
Threat Intelligence

Attack Classes Detected

CloudShield identifies four major categories of cyber attacks, each posing unique risks to cloud infrastructure.

💥

DoS Attacks

Denial-of-Service attacks flood network resources, making systems unavailable to legitimate users through traffic overload.

Severity: Critical
🔍

Probe Attacks

Surveillance and scanning attacks that gather information about network vulnerabilities for future exploitation.

Severity: Moderate
🔓

R2L Attacks

Remote-to-Local attacks where unauthorized remote access is gained to a local machine, bypassing authentication.

Severity: High
👑

U2R Attacks

User-to-Root attacks escalate privileges from a normal user account to root/admin, taking full system control.

Severity: Critical
